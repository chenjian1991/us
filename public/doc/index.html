    Introduction

    Welcome to the 55 Trade API

    Currently, we support Websocket and RESTful API for Market Data retrival.

    Endpoints

    We offer both Websocket (Stomp) and RESTful Endpoints.

    RESTFul Endpoint

    http://api.55.trade/v1

    WebSocket Endpoint

    http://ws.55.trade/v1/exchange/marketdata

    Market Data orderbook Snapshot

    Get Whole Snapshot for symbol

    curl "http://api.55.trade/orderbook/BTCUSD"
  -H "Authorization: DeveloperKey"

    The above command returns JSON structured like this:

    [
    {
      "asks":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
      "bids":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
      "provider":"BINANCE",
      "updateAt":1532994078139
    },
    {
      "asks":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
      "bids":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
      "provider":"HUOBI",
      "updateAt":1532994078139
    }
]

    This endpoint retrieves all venues top five orderbook market data snapshot.

    HTTP Request via RESTful

    GET /orderbook/<SymbolPair>

    Subscribtion to get real-time update via WebSocket

    /topic/orderbook/<SymbolPair>

    Query Parameters

    Parameter Nullable Description
    SymbolPair false The symbol pair that you want to take a snapshot from.
    Remember — All requests made to Unsupported Symbol Pairs would cause the endpoint to return an error code!

    Get a Specific Venue's Top 5 orderbook for a specific symbol

    curl "http://api.55.trade/orderbook/BTCUSD/BINANCE"
  -H "Authorization: DeveloperKey"

    The above command returns JSON structured like this:

    {
  "asks":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
  "bids":[{"qty":1.00,"price":2.00, "priceWithFee":2.11}],
  "provider":"BINANCE",
  "updateAt":1532994078139
}

    This endpoint retrieves a specific top 5 orderbook for specific venue and symbol pair.

    All our websocket endpoints only support StompClient <Aka, SockJS over Websocket> be sure to using correct library.

    HTTP Request via RESTful

    GET /orderbook/<SymbolPair>/<VenueName>

    Subscribtion to get real-time update via WebSocket

    /topic/orderbook/<SymbolPair>/<VenueName>

    URL Parameters

    Parameter Nullable Description
    SymbolPair false The symbol pair that you want to take a snapshot from.
    VenueName false The venue name that you want to take a snapshot from.

    Errors

    This error section is stored to describe possible error codes returned by our services.

    The 55 Trade API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The resources requested is hidden for administrators only.
    404 Not Found -- The specified resources could not be found.
    405 Method Not Allowed -- You tried to access a resource with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The resources requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many resources! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
